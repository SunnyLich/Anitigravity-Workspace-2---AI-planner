# Architecture Decision Records (ADR)

Use ADRs for decisions that materially affect architecture, dependencies, or long-term operations.

## Rules
1. One decision per file.
2. Keep status explicit: `Proposed`, `Accepted`, `Superseded`.
3. Record alternatives and trade-offs.
4. Link implementation PR/commit once executed.

## Naming
- `0001-short-title.md`, `0002-short-title.md`, etc.
