# Product Roadmap

Updated: 2026-03-09

## Goal
Build a practical London trip planner that is reliable under real-world constraints (time windows, user priorities, and transport mode realities).

## Now (P0)
1. Opening-hours-aware itinerary feasibility.
2. Max-priority-within-budget solver hardening.
3. Real transit provider selection and adapter foundation.

## Next (P1)
1. Replace mock routing fallback with OSRM.
2. Improve search ranking (fuzzy + weighted relevance).
3. Improve POI detail clarity and travel-method state feedback.
4. Expand custom node editing beyond rename.

## Later (P2)
1. Settings/help panel.
2. Additional onboarding and diagnostics in-app.

## Non-goals (current cycle)
1. New major UI surfaces beyond required planner and itinerary workflows.
2. Overhauling map stack or framework migration.
3. Feature expansion outside London-validated behavior until core constraints are stable.
