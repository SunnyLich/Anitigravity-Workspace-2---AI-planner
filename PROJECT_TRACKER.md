# Project Planning Index

Updated: 2026-03-12

This file is the navigation hub for planning and implementation context.

## Recommended reading order

1. Start with `README.md` for the public product summary.
2. Use `docs/roadmap.md` for near-term product direction.
3. Use `PROJECT_TRACKER.md` for implementation priorities and repo-quality follow-through.
4. Use the machine-readable planning files when working directly from backlog data.

## Canonical planning files

- Product roadmap: `docs/roadmap.md`
- Working tracker: `PROJECT_TRACKER.md`
- Workstream status: `planning/workstreams.yaml`
- Backlog tasks: `planning/backlog.json`
- Architecture context: `docs/architecture.md`
- Engineering onboarding: `docs/onboarding.md`
- Change history: `docs/changelog.md`
- Architecture decisions: `docs/adr/`

## Scope note

The README should describe what the app demonstrably does today.
The roadmap should describe what is next.
The tracker should describe what still needs execution effort.
